## 二、模块划分

### 2.1 RAG 对话模块

提供基于知识库的问答能力，支持文档检索和智能回答。

| API 路径 | HTTP 方法 | 功能描述 |
| :--- | :--- | :--- |
| `/api/rag/init-knowledge-base` | POST | 初始化/重建知识库向量库 |
| `/api/rag/chat` | POST | RAG 智能问答接口 |

### 2.2 Agent 作业管理模块

提供作业管理对话能力，支持添加、查询、更新作业状态等操作。

| API 路径 | HTTP 方法 | 功能描述 |
| :--- | :--- | :--- |
| `/api/agent/chat` | POST | Agent 作业管理对话接口 |

### 2.3 文件上传模块

提供文件上传功能，支持 txt、docx、pdf 格式文件上传到知识库。

| API 路径 | HTTP 方法 | 功能描述 |
| :--- | :--- | :--- |
| `/api/upload/file` | POST | 上传文件到服务器（不更新知识库） |
| `/api/upload/file-to-knowledge` | POST | 上传文件并更新知识库 |
| `/api/upload/knowledge-files` | GET | 获取知识库文件列表 |
| `/api/upload/knowledge-file/{filename}` | DELETE | 删除知识库中的文件 |

## 三、RAG 对话模块接口

### 3.1 初始化知识库

**接口地址**: `POST /api/rag/init-knowledge-base`

**功能描述**: 重新加载知识库文档并构建向量索引，用于知识库更新后重建索引。

**请求参数**: 无

**成功响应** (HTTP 200):

```json
{
    "message": "知识库构建成功"
}
```

**失败响应** (HTTP 500):

```json
{
    "detail": "知识库构建失败: <具体错误信息>"
}
```

**对应代码**: [app/api/rag/controller.py#L13-L23](file:///d:\Desktop\learning_assistant\app\api\rag\controller.py#L13-L23)

---

### 3.2 RAG 智能问答

**接口地址**: `POST /api/rag/chat`

**功能描述**: 根据用户提问，从知识库中检索相关文档并生成回答。

**请求体**:

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `question` | string | 是 | 用户的提问内容，不能为空 |
| `chat_name` | string | 是 | 对话名称，用于区分不同对话 |

```json
{
    "question": "什么是机器学习？",
    "chat_name": "AI基础学习"
}
```

**成功响应** (HTTP 200):

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `answer` | string | AI 结合知识库给出的完整回答 |
| `status` | string | 状态标识，成功为 "success" |

```json
{
    "answer": "机器学习是人工智能的一个分支...",
    "status": "success"
}
```

**失败响应**:

- **HTTP 400**: 提问内容为空

```json
{
    "detail": "提问内容不能为空"
}
```

- **HTTP 500**: 服务内部异常

```json
{
    "detail": "AI 问答服务异常: <具体错误信息>"
}
```

**对应代码**: [app/api/rag/controller.py#L29-L46](file:///d:\Desktop\learning_assistant\app\api\rag\controller.py#L29-L46)、[app/services/ai/rag/RAG.py](file:///d:\Desktop\learning_assistant\app\services\ai\rag\RAG.py)

## 四、Agent 作业管理模块接口

### 4.1 Agent 对话接口

**接口地址**: `POST /api/agent/chat`

**功能描述**: 基于大语言模型的作业管理对话接口，支持自然语言交互。

**请求体**:

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `question` | string | 是 | 用户的提问内容，不能为空 |
| `chat_name` | string | 是 | 对话名称，用于区分不同对话 |
| `chat_history` | array | 否 | 历史对话记录，用于上下文理解 |

```json
{
    "question": "帮我添加一项数学作业，截止日期是2026-06-15",
    "chat_name": "作业管理",
    "chat_history": []
}
```

**成功响应** (HTTP 200):

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `answer` | string | Agent 返回的回答内容 |
| `status` | string | 状态标识，成功为 "success" |

```json
{
    "answer": "好的，已为您添加数学作业，截止日期为2026年6月15日。",
    "status": "success"
}
```

**失败响应**:

- **HTTP 400**: 提问内容为空

```json
{
    "detail": "提问内容不能为空"
}
```

- **HTTP 500**: 服务内部异常

```json
{
    "detail": "Agent 服务异常: <具体错误信息>"
}
```

**对应代码**: [app/api/agent/controller.py](file:///d:\Desktop\learning_assistant\app\api\agent\controller.py)、[app/services/ai/agent/homework_agent.py](file:///d:\Desktop\learning_assistant\app\services\ai\agent\homework_agent.py)

---

### 4.2 Agent 支持功能

#### 4.2.1 作业管理能力

| 功能类型 | 示例指令 |
| :--- | :--- |
| 添加作业 | "帮我添加一项数学作业，截止日期是明天" |
| 查询全部作业 | "我有哪些作业？" |
| 查询未完成作业 | "我还有哪些作业没做？" |
| 完成作业 | "我已经完成了数学作业" |
| 学习计划 | "帮我安排一下学习计划" |

#### 4.2.2 时间处理能力

支持自然语言时间解析，自动转换为标准日期格式：

- "明天" → `YYYY-MM-DD`（明日日期）
- "这周五" → `YYYY-MM-DD`（本周周五日期）
- "下周" → 自动计算下周日期

#### 4.2.3 作业状态说明

| 状态值 | 说明 |
| :--- | :--- |
| `pending` | 作业未开始 |
| `completed` | 作业已完成 |

## 五、文件上传模块接口

### 5.1 上传文件（不更新知识库）

**接口地址**: `POST /api/upload/file`

**功能描述**: 上传文件到服务器，但不自动更新知识库索引。如需更新知识库，请使用 `/api/upload/file-to-knowledge`。

**请求体**: `multipart/form-data`

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `files` | File | 是 | 要上传的文件列表，支持多选 |

**支持的文件类型**: `.txt`, `.docx`, `.pdf`

**成功响应** (HTTP 200):

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `message` | string | 上传结果提示信息 |
| `file_names` | array | 成功上传的文件名列表 |

```json
{
    "message": "成功上传 2 个文件",
    "file_names": ["document1.txt", "report.pdf"]
}
```

**失败响应**:

- **HTTP 400**: 未选择文件或文件类型不支持

```json
{
    "detail": "请选择要上传的文件"
}
```

```json
{
    "detail": "文件类型不支持: example.jpg。仅支持 txt、docx、pdf 格式"
}
```

- **HTTP 500**: 文件上传失败

```json
{
    "detail": "文件 example.txt 上传失败: <具体错误信息>"
}
```

**对应代码**: [app/api/upload/controller.py](file:///d:\Desktop\learning_assistant\app\api\upload\controller.py)

### 5.2 上传文件并更新知识库

**接口地址**: `POST /api/upload/file-to-knowledge`

**功能描述**: 上传文件到服务器，并自动更新知识库向量索引。

**请求体**: `multipart/form-data`

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `files` | File | 是 | 要上传的文件列表，支持多选 |

**支持的文件类型**: `.txt`, `.docx`, `.pdf`

**成功响应** (HTTP 200):

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `message` | string | 上传结果和知识库更新提示信息 |
| `file_names` | array | 成功上传的文件名列表 |

```json
{
    "message": "知识库构建成功，共处理了 15 个文本切片，成功上传 2 个文件",
    "file_names": ["document1.txt", "report.pdf"]
}
```

**失败响应**:

- **HTTP 400**: 未选择文件或文件类型不支持

```json
{
    "detail": "请选择要上传的文件"
}
```

- **HTTP 500**: 文件上传或知识库初始化失败

```json
{
    "detail": "知识库初始化失败: <具体错误信息>"
}
```

**对应代码**: [app/api/upload/controller.py](file:///d:\Desktop\learning_assistant\app\api\upload\controller.py)

### 5.4 删除知识库文件

**接口地址**: `DELETE /api/upload/knowledge-file/{filename}`

**功能描述**: 删除知识库中的指定文件，并自动更新知识库索引。

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `filename` | string | 是 | 要删除的文件名 |

**成功响应** (HTTP 200):

```json
{
    "message": "文件 document1.txt 删除成功，知识库已更新"
}
```

**失败响应**:

- **HTTP 400**: 文件类型不支持

```json
{
    "detail": "文件类型不支持"
}
```

- **HTTP 404**: 文件不存在

```json
{
    "detail": "文件不存在: document1.txt"
}
```

- **HTTP 500**: 删除失败

```json
{
    "detail": "删除文件失败: <具体错误信息>"
}
```

## 六、错误码说明

| HTTP 状态码 | 含义 | 触发场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 接口正常执行完成 |
| 400 | 请求参数错误 | 缺少必填参数、文件类型不支持或参数格式错误 |
| 404 | 资源不存在 | 请求的文件或资源不存在 |
| 500 | 服务器内部错误 | 服务执行过程中发生异常 |

## 九、调用示例

### 9.1 调用 RAG 问答接口

```bash
curl -X POST http://localhost:8000/api/rag/chat \
  -H "Content-Type: application/json" \
  -d '{"question": "什么是深度学习？", "chat_name": "AI学习"}'
```

### 9.2 调用 Agent 作业管理接口

```bash
curl -X POST http://localhost:8000/api/agent/chat \
  -H "Content-Type: application/json" \
  -d '{"question": "帮我查看所有作业", "chat_name": "作业管理"}'
```

### 9.3 初始化知识库

```bash
curl -X POST http://localhost:8000/api/rag/init-knowledge-base
```

### 9.4 上传文件到知识库

```bash
curl -X POST http://localhost:8000/api/upload/file-to-knowledge \
  -F "files=@document1.txt" \
  -F "files=@report.pdf"
```

### 9.5 获取知识库文件列表

```bash
curl -X GET http://localhost:8000/api/upload/knowledge-files
```

### 9.6 删除知识库文件

```bash
curl -X DELETE http://localhost:8000/api/upload/knowledge-file/document1.txt
```
